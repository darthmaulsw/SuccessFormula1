# SuccessFormula1 — 24-Hour Hackathon Build Plan
# YHack 2026 | Start: 3:00 PM Saturday | Race: 1:00 AM Sunday ET | Demo: ~9:00 AM Sunday

==============================================================================
CONTEXT
==============================================================================

SuccessFormula1 is a real-time F1 race prediction dashboard targeting four
YHack prize tracks: Polymarket, K2 Think V2, Best UI/UX, and Most Creative.
The Japanese GP at Suzuka starts at 1 AM ET — the system must be fully live
and demo-ready before the race starts. Two people, 10 hours to pre-race,
~8 more hours to polish and demo. The repo is currently empty.

==============================================================================
🚨 3 CRITICAL PATH ITEMS THAT COULD KILL THE PROJECT
==============================================================================

1. MODAL INFERENCE ENDPOINT LATENCY / SETUP FAILURES
   Why: Every live prediction depends on Modal. Cold starts or auth failures
        kill the entire ML pipeline.
   De-risk: Start Modal setup in the first 30 minutes. Train model locally
            first, THEN migrate to Modal. Use @modal.web_endpoint. Keep a
            local fallback pickle for FastAPI to load directly.

2. OPENF1 LIVE DATA AVAILABILITY DURING RACE
   Why: OpenF1 may lag, rate-limit, or have gaps. Need fresh lap data ~90s.
   De-risk: Pre-test all endpoints tonight before the race. Cache last-known
            state so frontend never goes blank. Build DEMO_MODE that replays
            pre-recorded 2024 Suzuka JSON if API goes cold.

3. K2 THINK V2 API ACCESS / LATENCY
   Why: K2 is a prize track — broken integration = lost prize track.
   De-risk: Test API credentials in first hour. K2 calls must be async,
            fire-and-forget. Fall back to templated strings if K2 is down.

==============================================================================
TECH STACK
==============================================================================

Layer             | Technology                    | Why
------------------|-------------------------------|---------------------------
Backend           | Python FastAPI + uvicorn      | Fast to build, async SSE
ML Training       | scikit-learn XGBoost + Modal  | Trains in minutes
ML Serving        | Modal @modal.web_endpoint     | Zero-infra inference
Live Data         | OpenF1 REST API               | Free, no auth
Historical Data   | FastF1                        | Battle-tested Suzuka data
Radio NLP         | Faster-Whisper + VADER        | tiny model, no GPU needed
LLM Insights      | K2 Think V2 70B               | Prize track requirement
Polymarket        | Gamma API (REST)              | Live odds, no WS needed
Frontend          | React + Vite + TailwindCSS    | Fast scaffold
Charts            | Recharts                      | Animated bar charts
Real-time         | Server-Sent Events (SSE)      | Simpler than WebSocket

    "pit_stops":           int,    # number of pit stops taken

    "vsc":                 int,    # 0/1 flag
    "radio_sentiment":     float,  # VADER compound score [-1, 1]
    "radio_pit_keyword":   int,    # 1 if pit/box in last 2 laps
    "position_change_3lap":int,    # positions gained/lost in 3 laps
    "gap_delta_3lap":      float,  # gap change last 3 laps (neg=gaining)
}
